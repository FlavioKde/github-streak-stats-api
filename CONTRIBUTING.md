# Contributing to GitHub Streak Stats API

Thank you for your interest in contributing to this project.

We want to make contributing as easy and transparent as possible.

## Code of Conduct

This project and everyone participating in it are governed by our
[Code of Conduct](CODE_OF_CONDUCT.md).

By participating, you are expected to uphold this code.

## Development Setup

Install dependencies:

```bash
npm install
```

Run tests:

```bash
npm run test
```

## Branch Workflow

Create a feature branch from `main`:

```bash
git checkout -b feature/my-feature
```

## Commit Convention

This project follows the [Conventional Commits](https://www.conventionalcommits.org/) specification.

Examples:

* `feat(api): add JSON endpoint`
* `fix(cache): correct cache invalidation`
* `docs(readme): update installation guide`
* `chore(ci): update workflow`

## Pull Requests

We actively welcome pull requests from the community.

Before opening a Pull Request:

* Ensure all tests pass.
* Update documentation when necessary.
* Update `CHANGELOG.md` if applicable.
* Follow the project's commit conventions.

All contributions must be submitted through Pull Requests.

## Issues

Please use the provided issue templates when reporting bugs or proposing new features:

* Bug Report
* Feature Request

Issue templates help us understand and process contributions more efficiently.
